# CPSWC 产品简介

**生产建设项目水土保持方案智能编制与审查平台**

---

## 一句话定位

CPSWC 是一个**确定性水土保持方案生产引擎** — 从结构化事实数据出发，经规则推导、自动计算、确定性投影，产出可追溯的报告全套文件。

**这不是 AI 写作工具。**

---

## 解决什么问题

水土保持方案编制目前面临三个核心痛点:

| 痛点 | 现状 | CPSWC 解决方式 |
|---|---|---|
| **数据散落，改一处漏十处** | 占地面积在总说明、占地表、补偿费、防治范围等十余处出现。手工改动极易遗漏 | 单一事实源驱动全文，改一处自动联动所有引用 |
| **审查退件率高** | 退件原因多为"数据不一致""表文不符"，而非写作质量问题 | 表、文、计算共享同一数据源，从机制上消除不一致 |
| **经验依赖重，新手难上手** | 哪些章节要写、哪些可跳过、目标值怎么查表，全靠编制人员经验 | 规则引擎自动推导义务清单，计算器自动查表出值 |

## 三大核心能力

### 1. 确定性生产 — 不是"生成"，是"投影"

```
结构化事实 (46 字段)
  → 规则引擎 (13 条义务，条件触发)
  → 3 个计算器 (补偿费 / 六率目标 / 弃渣场级别)
  → 25 个叙述模板 + 13 张表格
  → 完整 submission package
```

相同输入 → 必然产出相同报告。每次运行结果可复现、可比较、可审计。

### 2. 改得动，查得清 — 事实变更追溯

修改一个事实字段后，系统自动追溯影响链:

- 哪些**计算器输出**变了 (如补偿费因面积变化而改变)
- 哪些**义务触发状态**翻转了 (如新增弃方 → 激活弃渣场章节)
- 哪些**正文段落**需要更新 (精确到行级 diff)
- 哪些**表格数据**受影响

实测案例: 将弃方从 0 改为 5 万m³ → 触发 3 条新义务、激活 3 个章节、修改 4 段正文、影响 2 张表。整个过程自动完成，无人工干预。

### 3. 可追溯闭环 — 每句话都有出处

报告中每段文字标注:
- **evidence_refs**: 数据来源 (如 `field.fact.land.total_area`)
- **source_rule_refs**: 法规依据 (如 `rule.gb50433_2018.sec_4_4_1`)

审查时可直接对照: 这句话的数据从哪来？依据哪条标准？

## 与"AI 写报告"的本质区别

| 维度 | AI 写作工具 | CPSWC |
|---|---|---|
| 内核 | 概率性语言模型 | 确定性规则引擎 |
| 可复现 | 每次结果不同 | 相同输入必然相同输出 |
| 数据一致性 | 无保证 | 单一数据源，机制保证 |
| 可追溯 | 无 | 每段话标注数据来源和法规依据 |
| 变更联动 | 需手工核对 | 自动传播，行级 diff 可视化 |
| 审查适配 | 无 | 义务清单对齐 SL/T 规范审查要点 |

## 适用场景

### 设计院 / 咨询公司

- **编制加速**: 结构化录入替代大量文字编写，首稿时间大幅缩短
- **质量保障**: 消除数据不一致这一最常见的退件原因
- **新手赋能**: 规则引擎自动判断义务清单和必填章节

### 审查机构

- **形式审查自动化**: 义务完整性、数据一致性可自动校验
- **影响链可视化**: 修改意见下达后可预见影响范围
- **审查效率提升**: 聚焦专业判断，减少低级数据核对

### 政务平台 / 主管部门

- **标准化管控**: 统一的事实模型和规则集确保报告格式合规
- **版本可追溯**: 每次提交含 SHA256 哈希，修改历史可审计
- **批量质检**: 接入规则引擎可实现报告批量机检

## 标准演示路径

```
1. 录入: project_intake_minimal.yaml → intake_validator → facts.json
2. 生成: facts.json → runtime → submission package (docx + workbench)
3. 变更: 修改 earthwork.spoil 0→5 万m³
4. 追溯: fact_diff → diff_workbench.html (7 个视图展示影响链)
5. 对比: 世维物流 vs 惠南智谷 — 同一引擎，不同项目，不同报告
```

## 技术要点

- 纯 Python 实现，无外部 AI/LLM 依赖
- 单一数据源 (facts.json) 驱动全链路
- Content-addressed snapshot (SHA256) 确保可审计
- 模块化架构: 规则集、计算器、模板均可独立扩展
- 已对齐: GB 50433-2018 / GB/T 50434-2018 / 水利部令 53 号 / 2023_177 审查要点
