## 一、费用五部分结构 (对齐样稿)

两份样稿的投资估算总表都按"五部分 + 预备费 + 补偿费"分项。本契约固定此结构为投资估算的**行骨架**：

| 序号 | 费用分项 | 归口类型 | v0 数据状态 |
|---|---|---|---|
| 一 | **第一部分 工程措施** | A+B (工程量输入 + 定额计算) | — (缺分项 facts) |
| 二 | **第二部分 植物措施** | A+B (同上) | — |
| 三 | **第三部分 监测措施** | A+B (监测点数/频次 + 费率计算) | — |
| 四 | **第四部分 临时措施** | A+B (临时工程量 + 定额计算) | — |
| 五 | **第五部分 独立费用** | B (按规定费率从一~四部分合计派生) | v1 可 live |
| I | 一至五部分合计 | 汇总 | — |
| II | 基本预备费 | B (一至五部分合计 × 预备费率) | v1 可 live |
| III | 水土保持补偿费 | **B (cal.compensation.fee)** | **v0 LIVE** |
| | **水土保持工程总投资** | 汇总 (I + II + III) | — (仅补偿费行 live) |

## 二、输入项边界

### A 类输入 (form_input) — 由设计院/用户提供

| 输入项 | 说明 | 来源标准 |
|---|---|---|
| 工程措施工程量 | 各防治分区的措施数量/面积/体积 | 措施布设设计 |
| 植物措施工程量 | 绿化面积/种植密度/苗木规格 | 措施布设设计 |
| 临时措施工程量 | 临时排水沟/拦挡/苫盖等数量 | 措施布设设计 |
| 监测点数/频次 | 各分区监测布设方案 | 监测规划 |
| 工程单价 | 各措施单位造价 | 当地定额 + 市场调查 |
| 材料预算价格 | 主要材料单价 | 当地市场价 |
| 机械台班费 | 施工机械台时费 | 当地定额 |

**准入红线**: 这些输入不允许系统自动填充、不允许从 total_investment 反推、不允许从样稿抄数字。缺则留 "—"。

### B 类可计算项 (calculator_generated) — 系统按规范费率计算

| 计算项 | 公式/来源 | authority_class |
|---|---|---|
| **补偿费** | cal.compensation.fee (已 LIVE) | official_approval (L4) |
| **独立费用** | 按规定费率从建安费/一~四部分合计派生 | regulation (L2) |
| **预备费** | 一至五部分合计 × 预备费率 (通常 10%) | regulation (L2) |

## 四、主体已列 vs 方案新增 拆分规则

样稿 2 (附表 1) 的列结构明确区分：

| 列 | 含义 |
|---|---|
| **方案新增投资** | 水土保持方案中**新增**的措施投资 |
| **主体已有水土保持投资** | 主体工程设计中**已包含**的水土保持功能措施投资 |
| **水土保持工程总投资** | 两者之和 |

**拆分规则**:
- 由设计院在 A 类输入中标注每项措施是"主体已列"还是"方案新增"
- 系统不自动判断拆分归属
- 缺标注时整项计入"方案新增"（保守原则）

## 六、金额格式与四舍五入规则

| 规则 | 说明 |
|---|---|
| 单位 | 正文表用 **万元** (精度 0.01); 附表可用 **元** |
| 四舍五入 | 分项四舍五入到小数点后 2 位, 合计从分项累加 (不从原始值重算) |
| 零值 | 显示 **0** (不显示 "—"; 0 表示"该项存在但金额为零") |
| 缺值 | 显示 **—** (表示"该项数据未提供/当前未建模") |
| 不适用 | 显示 **/** (表示"该项目不适用此费用项") |

---

## 七、空值三态语义 (Step 16 红线)

| 显示 | 含义 | 何时使用 |
|---|---|---|
| `—` | 数据未提供 / 当前未建模 | v0 缺分项 facts 时 |
| `0` 或 `0.00` | 明确为零 | 项目确实无此费用项但已确认 |
| `/` | 该项目不适用 | 如"表土保护率"对无表土项目 |

**禁令**: 不允许用 `0` 代替 `—`, 不允许用 `—` 代替 `/`。
